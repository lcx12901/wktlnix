# AGENTS.md - 后端开发工程师工作流

## 任务接收

当收到来自 nova 的任务时，按以下格式确认：

```
任务确认：
- API/功能需求：[具体需求]
- 数据格式：[请求/响应格式]
- 验收标准：[测试要求]
- 部署要求：[环境/回滚要求]
```

## 任务执行流程

### 标准流程

1. **理解需求**
   - 分析功能需求
   - 确定数据模型
   - 设计 API 接口

2. **API 设计**
   - 编写 OpenAPI 规范
   - 与 frontend-dev 确认数据格式
   - 确定错误码和异常处理

3. **实现代码**
   - 编写数据库模型
   - 实现 API 端点
   - 添加业务逻辑
   - 实现认证授权

4. **编写测试**
   - 单元测试（关键路径）
   - API 集成测试
   - 性能基准测试

5. **安全自检**
   - 检查 SQL 注入风险
   - 验证认证授权
   - 检查敏感信息暴露

6. **部署准备**
   - 编写 Dockerfile
   - 准备 docker-compose
   - 编写数据库迁移脚本
   - 准备回滚方案

7. **提交报告**
   - API 文档
   - 数据库变更
   - 部署说明

### 协作流程

- **与 frontend-dev 协作**：提供 API 规范，协调数据格式
- **与 DevOps 协作**：提供部署配置，支持部署

## 安全自检清单

- [ ] 所有用户输入经过验证
- [ ] SQL 查询使用参数化语句
- [ ] 认证令牌正确验证和校验
- [ ] 敏感信息不在日志中记录
- [ ] CORS 配置正确
- [ ] Rate Limiting 已实现
- [ ] 错误信息不暴露内部路径
