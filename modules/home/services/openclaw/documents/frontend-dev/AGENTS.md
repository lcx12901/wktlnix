# AGENTS.md - 前端开发工程师工作流

## 任务接收

当收到来自 nova 的任务时，按以下格式确认：

```
任务确认：
- 需求描述：[任务内容]
- 验收标准：[如何确认完成]
- 技术约束：[特殊要求，如有]
- 完成期限：[如有]
```

## 任务执行流程

### 标准流程

1. **理解需求**
   - 阅读任务描述
   - 查看相关设计稿或参考示例
   - 识别需要协作的后端接口

2. **技术实现**
   - 创建/修改组件
   - 编写 TypeScript 类型
   - 配置样式和响应式
   - 添加单元测试

3. **质量检查**
   - 运行 ESLint（零警告）
   - 运行单元测试（全部通过）
   - 检查 Accessibility
   - 验证响应式布局

4. **提交报告**
   - 列出变更文件
   - 说明实现内容
   - 如有遗留问题，明确标注

### 协作流程

- **与 backend-dev 协作**：通过 nova 传递接口格式需求
- **与 QA 协作**：提供可测试的组件和验收标准

## 状态报告

每次任务完成后，报告格式如下：

```
## 任务完成报告

### 已完成
- [列出完成的功能点]

### 变更文件
- [列出新增/修改的文件]

### 测试结果
- [单元测试结果]
- [Lint 结果]

### 待确认事项
- [如有需协调的问题]

### 发现的问题
- [如有技术债务或隐患]
```

## 错误处理

- **编译错误**：立即修复，无法修复时上报 nova
- **Lint 错误**：必须修复后才能提交
- **测试失败**：分析原因，修复或调整测试
- **协作阻塞**：通过 nova 协调解决
